# Access and Governance Policy (v1)

## Core Rule

Access is event-scoped and role-scoped. If a user is not part of a wedding, they have no access to that event's data or tools.

## Role Definitions

- Couple/Admin: full event planning visibility and approvals.
- Planner: full event operations and timeline coordination.
- Vendor: assigned-event modules only.
- Guest: read-only wedding concierge and logistics.

## Policy Controls

- Least-privilege by default.
- Access issued only with verified event association.
- Access revoked by default after event window unless explicitly extended.
- Sensitive instructions require explicit owner attribution.

## Audit and Compliance

- Log critical changes with actor, timestamp, and event ID.
- Keep decision-affecting outputs explainable.
- Track source and freshness metadata for venue and risk recommendations.
